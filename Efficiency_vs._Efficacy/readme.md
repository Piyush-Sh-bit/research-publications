# EEP Benchmark: Efficiency vs. Efficacy for Document Intelligence

**A Multi-Dimensional Benchmarking Framework for Evaluating Multimodal Large Language Models (MLLMs) Across Vision-Language Tasks in Zero-Shot and Fine-Tuned Settings.**

This repository contains the evaluation metrics, simulation codebase, and hardware-profiling scripts required to reproduce the Efficiency-Efficacy Pareto (EEP) benchmark analysis presented in our paper. 

---

## 🌟 The Efficiency-Efficacy Pareto (EEP) Framework

Existing benchmarks evaluate *only* semantic accuracy. However, practical industrial deployments are stringently constrained by compute, memory, and latency budgets. The EEP Framework is the **first-of-its-kind** metric that unifies:
1. **Computational throughput (TPS)**
2. **Peak memory utilization (VRAM in GB)**
3. **End-to-end generation latency ($L_s$ in seconds)**
4. **Task efficacy ($\eta$)**

### The $S_{EEP}$ Formulation:
$$S_{EEP} = \frac{\eta_{ANLS} \cdot \ln\bigl(1 + \frac{\Gamma}{\Omega}\bigr)}{(1 + L_{s}) \cdot \Psi}$$

## 📂 Repository Structure

- `code.ipynb` - The primary Jupyter Notebook containing the Monte Carlo simulation logic mapped to an NVIDIA A40 hardware profile, along with the visualization logic for generating all Pareto frontiers and error distributions found in the paper.
- `eep_benchmark/` - Core Python module containing:
  - `dataset.py`: Simulated dataset aggregators for processing the Scanned Images Dataset (SID).
  - `engine.py`: Baseline profiling engine supporting zero-shot and simulated supervised fine-tuning loops.
  - `metrics.py`: Implementation of the full unified EEP calculus and normalized ANLS.
  - `run_inference.py`: CLI script for generating local hardware latency measurements.
- `fig_*.png` - The primary analysis figures generated by our scripts.

## 🚀 Quickstart: Reproducing $S_{EEP}$ Scores

1. **Clone the repository:**
```bash
git clone https://github.com/AnonymousScholar/EEP-Benchmark-Replicable.git
cd EEP-Benchmark-Replicable
```

2. **Install dependencies:**
We recommend a standard Python 3.10+ data science stack.
```bash
pip install numpy pandas matplotlib seaborn jupyter scikit-learn
```

3. **Run the Analysis:**
Launch `code.ipynb` locally and execute all cells. The notebook pulls defined baseline measurements (documented inside the notebook) and executes the Monte Carlo simulation to project zero-shot baseline execution and simulated fine-tuning metrics. It will successfully re-generate the Pareto frontier maps over 18,420 simulated document evaluations.

## 📊 Dataset (SID)
The curated Scanned Images Dataset (SID) utilizes public components (DocVQA, RVL-CDIP) along with customized synthetic traces. The `dataset.py` mock class includes logic defining the distribution of our 3 layout-complexity tiers alongside multilingual script distributions (Latin, Devanagari, Chinese, Arabic, Cyrillic).
